> Для кого: человек, который **впервые** открывает проект и не знает ни ледников, ни машинного обучения, ни программирования на продвинутом уровне.

## 1. Одним абзацем — что это?

**GlacierNET-KZ** — это программа (и научный проект), которая **смотрит на спутниковые снимки гор Казахстана** и **автоматически находит ледники** на них. Потом она **считает, как уменьшалась площадь льда** с 2000 по 2020 год и **прогнозирует**, что будет к 2050 году.

Главная идея: вместо того чтобы вручную обводить ледники на карте (долго и дорого), нейросеть **U-Net** делает это за секунды с точностью **F1 ≈ 0.88** (очень хорошо для науки).

**Где это нужно:** ледники Заилийского (Илийского) Алатау кормят реки, от которых зависит вода для **Алматы** (~2.4 млн человек). Лёд тает — через десятилетия меньше воды.

**Кто автор:** Dulat Nurlanuly · open-source проект под лицензией **MIT** (можно свободно изучать и использовать код).

## 2. Словарь простыми словами

| Термин | Что это значит |
|--------|----------------|
| **Ледник** | Масса льда в горах, которая медленно движется вниз по склону |
| **Спутниковый снимок** | Фото Земли с космоса (как Google Maps, но с научными каналами) |
| **Sentinel-2** | Европейский спутник, снимки с 2015 г., разрешение ~10 м |
| **Landsat** | Американский спутник, архив с 1970-х, используем 2000–2013 |
| **GeoTIFF (.tif)** | Файл-карта с координатами и несколькими «слоями» (каналами) |
| **NDSI** | Простой индекс «насколько похоже на снег/лёд» по цвету пикселя |
| **U-Net** | Нейросеть для сегментации: каждый пиксель — лёд или не лёд |
| **Маска** | Чёрно-белая картинка: белое = ледник, чёрное = всё остальное |
| **RGI** | Мировой каталог контуров ледников (эталонные границы) |
| **WGMS / Туюксу** | Наземные измерения ледника Туюксу с 1957 г. — для проверки |
| **F1 / IoU** | Числа качества модели: чем ближе к 1, тем лучше |
| **Патч** | Кусок снимка 256×256 пикселей для обучения нейросети |
| **FastAPI** | Серверная программа, отдаёт данные через интернет (REST API) |
| **Next.js** | Современный веб-сайт (дашборд) |
| **Gradio** | Простая веб-страница «загрузи файл — получи результат» |
| **Docker** | Упаковка всех сервисов в контейнеры для запуска одной командой |

## 3. Что проект умеет (возможности)

### Научное ядро
- Скачать снимки через **Google Earth Engine** (ноутбук 01)
- Построить маски и нарезать данные на патчи (ноутбук 02)
- Сравнить **NDSI**, **Random Forest** и **U-Net** (ноутбуки 03–04)
- Посчитать площадь ледников по годам и **прогноз до 2050** (ноутбук 05)
- Нарисовать графики для статьи и постера (ноутбук 06)

### Веб и API
- **Дашборд** — карты, графики, загрузка снимков (EN / RU / KK)
- **API** — программный доступ: сегментация, тренд, экспорт, история
- **Gradio demo** — быстрая демонстрация для жюри
- **MCP** — инструменты для ИИ-ассистентов (Claude, Cursor)

### Результаты (реальные данные)
| Показатель | Значение |
|------------|----------|
| Точность U-Net (F1) | **0.876** |
| Потеря площади 2000–2020 | **−22.4%** (−129.5 км²) |
| Скорость таяния | **−12.7 км²/год** |
| Прогноз на 2050 | **~350 км²** останется |

### `src/` — мозг машинного обучения (39 модулей)

Центральный Python-код. Всё, что связано с нейросетями и спутниковыми данными.

| Файл | Простое описание |
|------|------------------|
| **config.py** | Все настройки: координаты, годы, размер патча, пути к папкам |
| **data_loader.py** | Читает GeoTIFF, считает спектральные индексы |
| **preprocessing.py** | Режет снимки на кусочки, делит train/val/test |
| **models.py** | U-Net, Attention U-Net, Random Forest, NDSI — главные модели |
| **metrics.py** | F1, IoU, тренд по годам, прогноз 2050, WGMS-валидация |
| **visualization.py** | Рисует карты и графики |
| **train.py** | Запуск обучения из терминала |
| **losses.py** | Функции потерь для нейросети (Dice, Focal, …) |
| **ensemble.py** | Объединение нескольких моделей в одну |
| **uncertainty.py** | Оценка «насколько модель уверена» (MC-Dropout) |
| **postprocessing.py** | Уточнение границ маски после нейросети |
| **spectral.py** | Дополнительные спектральные индексы |
| **augmentation.py** | Искусственное увеличение данных (повороты, шум) |
| **segmentation_models.py** | Альтернативные архитектуры (DeepLab, PSPNet, …) |
| **vision_transformer.py** | Эксперимент: ViT вместо U-Net |
| **diffusion_model.py** | Эксперимент: диффузионные модели |
| **federated_learning.py** | Эксперимент: обучение без передачи данных |
| **…остальные…** | Продвинутые/экспериментальные модули (NAS, GNN, …) |

> **Важно:** для базового воспроизводимого пайплайна достаточно **config, data_loader, preprocessing, models, metrics, visualization**. Остальное — расширения и эксперименты.

## 6. С чего начать новичку

### Шаг 0 — понять идею (15 минут)
1. Прочитать **README.md**
2. Открыть **http://localhost:8080/hub** (если запущено) или HuggingFace demo
3. Посмотреть картинки в `results/figures/`

### Шаг 1 — установка (30–60 минут)
```bash
conda create -n glaciers python=3.10
conda activate glaciers
conda install -c conda-forge gdal rasterio geopandas
pip install -r requirements.txt
pip install -e ".[dev,api]"
```

### Шаг 2 — проверка без спутников (5 минут)
```bash
python notebooks/_synthetic_smoke_test.py
python notebooks/_unet_smoke_test.py
pytest tests/ -q -m "not experimental"
```

### Шаг 3 — запуск всего стека (1 команда)
```bash
./scripts/start.sh
# → http://localhost:8080/hub
```

### Шаг 4 — полный пайплайн (нужен Google Earth Engine)
```bash
earthengine authenticate
jupyter lab notebooks/
# Выполнить 01 → 06 по порядку
```

## 8. Частые вопросы

**Почему так много файлов?**  
Проект вырос в полноценную платформу: ML + API + сайт + демо + тесты + документация.

**Нужен ли GPU?**  
Для обучения U-Net — желательно. Для инференса (предсказания) и веб-интерфейса — хватит CPU.

**Где живут большие данные?**  
В `data/` и `models/` — они не в GitHub из-за размера.

**Какой файл главный?**  
Для науки: `src/models.py` + ноутбуки. Для веба: `glacierkz-api/app/main.py` + `glacierkz-web/src/app/page.tsx`.

**Как остановить всё?**  
`./scripts/start.sh --stop`
